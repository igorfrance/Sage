Internationalization

Internationalization is useful when your website speaks more than one language, and/or presents different content for different locales.

The internationalization in Sage involves taking an XML resource and making separate physical copies of it for each of the locales configured for project, where each copy can be as different as required, and Sage provides plenty of ways to branch content based on locale.

The internationalization techniques Sage supports are:

  • Phrase substitution

    Phrase substitution is the process of substituting text with content from the dictionary files. The entities used for phrase substitution are intl:phrase element and intl:phrase(id) syntax when used within attribute values.

  • Variable substitution

    Similar to phrases, variable substitution is the process of substituting text with variables defined earlier in the document. Variables can be configured per locale, with an optional default value. The entities used for variable substitution are intl:variable element and intl:variable(id) syntax when used within attribute values.

  • Conditional rendering

    When the differences between locales are structural, we can branch and selectively render content on the basis of the locale by using intl:localize/intl:if/intl:else elements, and the @intl:locale attribute.

  • Separate files

    If differences between localized versions of the file are big enough to warrant a separate file, the locale-specific version of the file simply needs to add the locale suffix to the name of the file; read more about it in the description of the XML localization.
