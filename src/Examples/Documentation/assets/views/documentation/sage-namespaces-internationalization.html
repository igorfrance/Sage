The internationalization namespace
The Internationalization namespace
Provides internationalization-related elements and attributes.
Namespace URI:
http://www.cycle99.com/schemas/sage/internationalization.xsd
Elements
intl:dictionary element
This element is a container for phrase elements in dictionary files that define language strings used throughout the project.
Example:
true Home Shop Contact F.A.Q.
Child elements:
intl:phrase element
Specifies a single phrase. Although the content will typically be plain text, this element may contain any number of XHTML and other XML elements. It may even contain references to other phrases and variables through the use of phrase[ref] and variable[ref] elements. They will all be resolved during internationalization.
Example:
true Home
Attributes
id
Specifies the unique id of the phrase. The phrase[ref] element refers to this id through it's ref attribute.
intl:variables element
This element may appear on several places where internationalization variables may be defined. These are (from low to high priority):
  • Project.config (global variables)
  • Category.config (category variables)
  • view document (view variables)
Example:
true http://shop.chew.com/pages/products/chewymango http://shop.chew.co.uk/pages/products/chewymango http://shop.chew.de/pages/products/chewymango http://shop.chew.nl/pages/products/chewymango http://shop.chew.fr/pages/products/chewymango http://www.showchewing.ca/pages/products/chewymango http://shop.chew.ru/on/imaginationware.store/chewymango http://shop.chew.co.kr/chewymango http://www.showchewing.com/chewymango http://shop.chew.jp/pc/item/chewymango
Child elements:
variable element

Specifies a single variable. The variable may contain reference to other phrase[ref] and variable[ref] elements, as they will all get resolved during internationalization.

The child value elements specify separate values for each locale. The order in which the values are resolved is controlled by dictionaryNames attribute defined in project configuration. Read more about it in internationalization variables.

Example:
false http://shop.chew.com/pages/products/chewymango http://shop.chew.co.uk/pages/products/chewymango http://shop.chew.de/pages/products/chewymango
Attributes
id
Specifies the unique id of the variable. The variable[ref] element refers to this id through it's ref attribute.
Child elements:
value element
Specifies the variable value for one locale.
Example:
http://shop.chew.co.uk/pages/products/chewymango
Attributes
locale
The name of the locale for which this value applies.
intl:variable[ref] element

Referes to a variable, providing a substitution placeholder for that variable. During internationalization, these elements get substituted with the value of that variable in each generated locale.

These elements can be placed anywhere in XML documents where the variable value is needed.

Attributes
ref
The id of the variable that this element refers to.
intl:phrase[ref] element

Referes to a phrase, providing a substitution placeholder for that phrase. During internationalization, these elements get substituted with the value of that phrase for each generated locale.

These elements can be placed anywhere in XML documents where the phrase value is needed.

Attributes
intl:locale attribute

Signifies that the element that this attribute appears on should only appear in the locale(s) specified by it. It can be attached to any element except the sage:include element.

The value of this attribute is a comma-separated list of names of locales in which the element to which this attribute is attached should appear; intl:locale="fr"

To reverse the logic of attribute (show in all locales except for the specified list), surround the list of locates within a not(); intl:locale="not(fr)"
