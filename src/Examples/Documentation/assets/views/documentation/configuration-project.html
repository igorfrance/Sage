Project configuration

The main project configuration file, Project.config, contains many settings that control how the application behaves during runtime.

Configuration files

There are three types of different project configuration files, with unique file and element names. The structure of these files is exactly the same (with the exception for extension configuration, which provides one additional element), but the usage and application is different.

Project.config

This is the main project configuration file that is specific to the current project. Any values not specified in this file will be used from system configuration file, if it exists. When a project initializes

System.config

This is another project configuration file, containing the default values. This configuration file is provided by Sage, and serves to provide defaults. This approach enables specific project configurations to be smaller, freeing them from having to specify everything explicitly.

Extension.config

The extension configuration file is actually a regular project configuration file that is modified and packed as Extension.config info the deliverable archive when an extension project is compiled into an extension. One additional element that an extension project's configuration file contains is the package element, that specifies how the extension should be packed into the archive.

Configuration elements

The following elements appear in any order within project, extension or system configuration files.
