Project configuration
Project configuration
Project configuration file stores all user configurable properties related to the project.
  • The name of the configuration file is Project.config
  • Sage first looks for a file with Sage defaults; System.config. The file should be in the working directory of the application. Working directory will typically be /bin.
  • Sage then tries to open Project.config file from the level above the working directory (../Project.config); failing that it then looks in the working directory (./Project.config).
  • Settings defined in Project.config will override equivalent settings from System.config.
  • Project.config can be divided in several files using sage-include elements.
  • The configuration file is optional; without it, project will initialize with the built-in defaults.
Reference
project element
The project element is the root element of the project configuration. It provides a few important category-and-internationalization-related attributes (defaultLocale, defaultCategory and sharedCategory) and development-related attributes (autoInternationalize and debugMode). The development attributes are typically controlled by the build script and need not be changed manually.
Example
Attributes
General
name
Specifies the name of the project.
Category and internationalization
defaultCategory
Sets the default category to use in multi-category projects where no specific category has been specified. The category with the specified name has to exist.
sharedCategory
Sets the name of the shared category to use in multi-category projects. The default value is 'shared'.
defaultLocale
Sets the default locale to use in multi-locale projects where no specific locale has been specified. The locale with the specified name has to exist.
Development
autoInternationalize
Controls weather Sage will automatically internationalize XML files that reference the internationalization namespace. Possible values are true or false.
debugMode
Controls how the system behaves in situations where additional debugging information may be emitted. In particular, determines whether merged scripts will be output as a single script or multiple scripts, as well as how certain XSLT errors are displayed. Possible values are true or false.
