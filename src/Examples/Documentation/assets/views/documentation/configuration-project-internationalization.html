Internationalization
If a project needs to be presented in multiple languages, possibly with different presentations and additional, replaced or removed content for each of those locales, it can be "internationalized". Internationalization in Sage is a process of branching content and presentation based on the locale. It enables projects that operate in multiple languages to integrate locale-based differences smoothly and naturally, and it is applied on several different levels. The locales (languages) that the project supports are defined in the internationalization section of the project configuration. Find out more.

Child elements
locale element
Defines a single locale
Example
Attributes
name
Specifies the name of the locale. Any name can be used here but typically this would be a value such as en or en-US.
dictionaryNames
Specifies the names of dictionaries that can be used with this locale, ordered by priority. See internationalization phrases.
resourceNames
Specifies the priority of name suffixes for locale-specific file overrides. See internationalization fallback.
Child elements
format element
Sets the cultture and date formatting options for the locale.
Example
Attributes
culture
Specifies the name of the CultureInfo instance to use with this locale.
shortDate
Specifies the short date format to use with this locale. This value is optional and should only be used if it is different than the format provided by this locale's CultureInfo instance. Default value is d, which is represents the date and time format string that is defined by this locale's CultureInfo DateTimeFormatInfo.ShortDatePattern property.
longDate
Specifies the long date format to use with this locale. This value is optional and should only be used if it is different than the format provided by this locale's CultureInfo instance. Default value is D, which is represents the date and time format string that is defined by this locale's CultureInfo DateTimeFormatInfo.LongDatePattern property.
