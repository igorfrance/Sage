Input document
The HTML documents you write are wrapped within a standard XML envelope before being served to the nearest XSLT processor for transformation.

Even though views are typically written simply as plain XHTML, they are processed and wrapped with additional XML by Sage before they are sent off for the final XSLT transform.

For a real-world example of the actual input document XML for this page, click here. Once you're on that page, you can use the plus ('+') icons to expand any collapsed nodes and drill deeper into the document structure.

  • sage:request

    Contains information about the current request.

    Attributes
    • @method
      The current HTTP method; typically GET or POST.
    • @localAddress
      The IP address of the local server.
    • @remoteAddress
      The IP address of the remote client.
    • @category
      The name of the current category
    • @locale
      The name of the current locale
    • @thread
      The name given to the thread of the request, used to identify this request's log messages in the main log file.
    • @developer
      Indicates whether the user making this request is considered a developer. Developers are granted a few privileges without having to log on. Developers are selected on the basis of the IP address, this is configured in the project configuration file. Possible values of this attribute are zero (0) and one (1).
    • @debug
      Indicates whether the current project is running in debug mode. When running in debug mode, modules can emit debugging information inline to aid with problem fixing. When not in debug mode, errors are typically caught and logged without showing the error to the end user. This value is configured in the project configuration file. Possible values of this attribute are zero (0) and one (1).
    Parent elements
    • sage:view
    Child elements
    • sage:path, sage:address, sage:useragent, sage:assembly, sage:querystring, sage:form, sage:cookies, sage:dateTime.
  • sage:path

    Provides the current paths that might be useful for the frontend.

    Attributes
    • @applicationPath
      The current application path, for instance; /myapp/.
    • @assetPath
      The path to the assets, taking the current category into account (if running a multi-category project), for instance; /myapp/assets/, or /myapp/assets/category1/.
    • @sharedAssetPath
      The path to the shared assets, applicable in multi-category scenarios, for instance; /myapp/assets/shared/.
    • @modulePath
      The modules directory path, for instance; /myapp/assets/lib/modules/.
    Parent elements
    • sage:request
    Child elements
    • None.
  • sage:address

    Provides information about the current URL and its constituent parts.

    Attributes
    • @basehref
      Fully qualified base URL for the current application, for instance; https://www.myserver.com/myapp/
    • @url
      The current, visible URL of the request, for instance; https://www.myserver.com/myapp/pages/mycoolpage.html?show=everything
    • @referrer
      The URL referrer string, if any.
    • @serverName
      The name of the current server, for instance; www.myserver.com.
    • @serverNameFull
      The name of the current server, prefixed with the current protocol, with server port number added (if other than 80 for http and 443 for https); https://www.myserver.com:444.
    • @scriptName
      The local part of the current URL; /myapp/pages/mycoolpage.html.
    • @scriptNameFull
      The local part of the current URL, with the quesry string included; /myapp/pages/mycoolpage.html?show=everything.
    • @querystring
      Only the querystring part of the URL, for instance; show=everything&page=45&order=desc
    Parent elements
    • sage:request
    Child elements
    • None.
  • sage:useragent

    Provides information about the current user agent.

    Attributes
    • @id
      The user agent ID. This is a copy of the value of HttpBrowserCapabilities.Type property, or in other words; the name and major (integer) version number of the browser. Examples: firefox11, chrome17 or ie9.
    • @name
      The name of the user agent. Examples: IE, Chrome or Firefox.
    • @version
      The version string of the user agent, for instance; 11.0.
    • @version.major
      The major part of the user agent version, for instance; 11.
    • @version.minor
      The minor part of the user agent version, for instance; 0.
    • @isCrawler
      If the user agent string indicates that the current user agent is a crawler (Googlebot, YahooSeeker, etc), this attribute's value will be 1; otherwise it will be 0.
    • @value
      The raw user agent string, for instance; Mozilla/5.0 (Windows NT 6.1; WOW64; rv:11.0) Gecko/20100101 Firefox/11.0
    Parent elements
    • sage:request
    Child elements
    • None.

  • sage:querystring

    This element provides read access to all POSTed form values contained in the request, in the form of attributes, where each value's name is the name of the attribute, and the attribute value is the posted value. If the request method is GET, this element will not be present. If there are no posted values at the time of the request, the element will have no attributes.

    Attributes
    • Dynamic list of attributes corresponding to the QueryString values.
    Parent elements
    • sage:request
    Child elements
    • None.
  • sage:form

    This element provides read access to all POSTed form values contained in the request, in the form of attributes, where each value's name is the name of the attribute, and the attribute value is the posted value. If the request method is GET, this element will not be present. If there are no posted values at the time of the request, the element will have no attributes.

    Attributes
    • Dynamic list of attributes corresponding to the POSTed values.
    Parent elements
    • sage:request
    Child elements
    • None.
  • sage:cookies

    This element provides read access to all cookies applicable for the request, in the form of attributes, where each cookie's name is the name of the attribute, and the attribute value is the value of that cookie. If there are no applicable cookies at the time of the request, the element will have no attributes.

    Attributes
    • Dynamic list of attributes corresponding to currently applicable cookies.
    Parent elements
    • sage:request
    Child elements
    • None.
  • sage:dateTime

    Contains attributes that describe the current date and time. By adding this element, these values become available to XSLT for further processing based on date and/or time, without having to resort to extension methods (as XSLT 1.0 has no built-in support for working with dates).

    Attributes
    • @date
      Contains the current date at the moment of the request, in the fixed format of dd-MM-yyyy
    • @time
      Contains the current date at the moment of the request, in the fixed format of hh:mm:ss
    • @day
      Contains the current date at the moment of the request, in the fixed format of dd
    • @month
      Contains the current month at the moment of the request, in the fixed format of MM
    • @year
      Contains the current year at the moment of the request, in the fixed format of yyyy
    • @hour
      Contains the current hour at the moment of the request, in the fixed format of hh
    • @minute
      Contains the current minute at the moment of the request, in the fixed format of mm
    • @second
      Contains the current second at the moment of the request, in the fixed format of ss
    Parent elements
    • sage:request
    Child elements
    • None.
  • sage:response

    Contains the response data for the current view; the view template (within sage:model element) and the resources used by the view template (within sage:resources element.

    Parent elements
    • sage:view
    Child elements
    • sage:resources, sage:model.
  • sage:resources

    This element is container for all resources reference by the view's modules and/or libraries. If there are no data, head or body resources, this element will not be present.

    Parent elements
    • sage:response
    Child elements
    • sage:data, sage:head, sage:body.
  • sage:data

    Contains the module and/or library document resources that should be loaded within this element, and hence be made available to XSLT processing (typically by the module that references it). If there are no data resources on the current view, this element will not be present.

    Parent elements
    • sage:resources.
    Child elements
    • Any.
  • sage:head

    Contains the module and/or library resources, such as scripts and styles referenced by the view or the modules contained on it, that should be loaded within the head of the HTML page. If there are no head resources on the current view, this element will not be present.

    The xhtml:link elements supported are stylesheets (rel=stylesheet) and shortcut icons (rel=icon).

    Parent elements
    • sage:resources.
    Child elements
    • xhtml:script, xhtml:link.
  • sage:body

    Contains the module and/or library resources, such as scripts and styles referenced by the view or the modules contained on it, that should be loaded within the body of the HTML page. If there are no body resources on the current view, this element will not be present.

    Parent elements
    • sage:resources.
    Child elements
    • xhtml:script, xhtml:link.

    Holds the X(HT)ML view configuration document that was used; typically, the XHTML file that represents the current view.
