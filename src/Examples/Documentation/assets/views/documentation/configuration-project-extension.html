Extension package
If the project is an extension project, this section can be used to define which of the project's assets and configuration sections to include in the extension zip archive. This enables clean deployment; the full project may contain various non-deliverable items; test views and files, unit tests, work in progress etc. This element specifies exactly which resources to include in the archive; anything else will not be touched.
package element

Defines how to package the project as an extension archive. Each of the child elements represents one group, and each group can contain any number of include and exclude elements that will be evaluated together into a set of selected items. First, include elements are evaluated one by one to create a selection of distinct items, after which exclude elements are evaluated one by one to determine which of the items to exclude from selection.

For binaries and assets elements the include and exclude selections refer to actual files; for all other elements they refer to the names of corresponding configuration sections that should be copied to the extension's project configuration file that will be generated (and later merged into the destination projects).
Example
~/bin/Sage.DevTools.dll ~/assets/modules ~/assets/scripts/lib ~/assets/views/dev * * *

Both the include and exclude elements have the same structure; they specify one of the three types of expression:

~/assets/scripts/*
~/assets/scripts
Represents all files and folders in ~/assets/scripts/.
~/assets/scripts/*.css
Represents all cssfiles in ~/assets/scripts/.
~/assets/scr*
Represents all files in ~/assets/ that start with scr.
Child elements
*/include
Specifies a single include expression.
Example
Include all files in ~/assets/scripts: ~/assets/scripts/*
Example
Include all binary assemblies in ~/bin: ~/bin/*.dll
Example
Include all modules whose name starts with documentation: documentation*
*/exclude
Specifies a single exclude expression.
Example
Exclude all files from ~/assets/modules/dev: ~/assets/modules/dev
Example
Exclude all links that start with internal: internal*
