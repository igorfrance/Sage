The Contextualization namespace
The Contextualization namespace
Provides elements that enable conditional rendering based on values in the current context.
Namespace URI:
http://www.cycle99.com/schemas/sage/contextualization.xsd
Elements
context:switch
Creates a switch block (similar to xsl:choose element), where the same property is evaluated for each child context:case element, and the contents of the first context:case element that evaluates to true is rendered, excluding all other siblings.

Property Description
Application The collection of application variables. Requires key to specify the item to get.
Query The collection of query string parameters. Requires key attribute.
RouteValues The colllection of current route values. Requires key attribute.
Session The collection of session variables. Requires key attribute.
ApplicationPath The virtual root path of the ASP.NET application on the server.
PhysicalApplicationPath The physical file-system path of the current application's root directory.
BaseHref The base href value (e.g. http://server:port/appname/") for requests within current context.
Category Gets the currently applicable category.
Locale Gets the currently applicable locale.
IsDeveloperRequest Gets a value indicating whether this context is running within a developer request.
ReferrerUrl Gets the URL of the client request that linked to the current URL.
UserAgentID Gets the id of the current user agent.
UserAgentType Gets the type of the current user agent.
key
If the property specified with the property attribute is a collection (Application, Session, Query, RouteValues), this attribute specifies the key within this dictionary to which the switch applies.
Child elements:
context:case
Defines a single possible match case within the switch it belongs to.
Example:
Attributes
test
The property value to test for. If the property value matches the value of this attribute, the block content will be rendered, replacing the whole context:switch element.
context:default
Defines the content to render when none of the sibling context:case elements evaluate to true.
Example:
