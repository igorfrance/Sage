Extensions projects
Extension projects are Sage projects packaged into archives and delivered to other Sage projects.

Developing

As a type of Sage project, extension projects can have any combination of controllers, modules, views and other resources. The extension project itself can be worked on independently and the extension project itself provides the text/example project of the extension.

The project configuration specifies which of the extensions components and resources should be copied into the deliverable archive. This allows extension developers to mix other work items, such as test views and specifications with the deliverable code, and filter them out in the compile step.

Packaging

The extension projects are packaged by the Sage.Build utility, invoked from the command line. Relative paths are considered relative to the working directory.

sage.build extension -source:<extension-project-path> -target:<path-to-archive>

Consuming

The extension archives are placed in the extensions directory of the target project. With each initialization of the application, the extension will be registered with the project.
